Compound Components

Compound
Components

Componentes que trabajan juntos como <Select> + <Option> o <Tabs> + <Tab>.

desliza para ver ejemplos
Rígido

Componente monolítico

<Tabs
  items={[...]}
  activeTab={0}
  onSelect={fn}
/>

No puedes personalizar cada pestaña. Todo se controla con props.

Flexible

Componentes compuestos

<Tabs>
  <TabList>
    <Tab>Productos</Tab>
    <Tab>Reviews</Tab>
  </TabList>
  <TabPanels>
    <TabPanel>...</TabPanel>
    <TabPanel>...</TabPanel>
  </TabPanels>
</Tabs>

Cada parte se personaliza libremente. El estado se comparte por contexto.

Internamente

Context como pegamento

const TabsContext = createContext();

function Tabs({ children }) {
  const [active, setActive] = useState(0);
  return (
    <TabsContext.Provider value={{active, setActive}}>
      {children}
    </TabsContext.Provider>
  );
}

Tab y TabPanel consumen el contexto para saber cuál está activo.

Ventajas

¿Por qué usarlo?

Personalización total
API declarativa
Código más limpio

Librerías como Chakra UI o Radix lo usan para dar máxima flexibilidad.

Tip rápido

Cuándo aplicarlo

Si tu componente tiene muchas partes
interdependientes (tabs, acordeones,
menús, selects) → Compound Components.

Resumen: divide la UI en piezas pequeñas que se comunican por contexto. El consumidor decide el orden y estilo.
